Welcome to Craft 1.1. Here’s what’s new.

Revamped tagging

Tags are no longer a sad little setting, only used by entries. They’ve been completely rethought, with maximum flexibility in mind.

  • You can create multiple sets of tags from Settings → Tags (called Tag Sets).
  • There’s a new Tags field type which can be used to add tags to entries, assets, users, global sets, and even other tags.
  • There’s a new craft.tags template variable for accessing all of your tags directly from the templates.

Note that if you had previously assigned any tags to your entries, those tags have been migrated over. However you will need to edit your sections’ Field Layouts to get the auto-created Tags field to appear in your entries’ edit pages.

Assets improvements

Assets have also seen a lot of love in this update.

  • It’s now possible to upload new assets directly from your Assets fields!
  • You can now edit your selected assets’ content directly from your Assets fields by double-clicking on them.
  • There’s a new craft.assets template variable for accessing all of your assets directly from the templates.

Multi-environment support

We’ve added a couple key features that make running the same Craft install in multiple environments much simpler.

  • Your config files can now define global, as well as per-environment config settings.
  • We’ve added a new ‘environmentVariables’ config setting which allows you to define variables which can be accessed from your Site URL and asset sources’ URL and Path settings.

Lots more!

With 62 additions and improvements, Craft 1.1 is our single biggest update yet. To see all that has changed, check out buildwithcraft.com/updates
